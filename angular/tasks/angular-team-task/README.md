# RS Angular Sprint

## 1. Project Goals

**Goals:**

- Learn the fundamentals of Angular by building a real application.
- Master key concepts: Components, Routing, Signals, Directives, Pipes, Forms, DI, Change Detection, HTTP, Testing.
- Gain experience in teamwork and code review.
- Demonstrate what you have learned over 4 sprints.

**Each team chooses one of three projects:**

| Project         | Description                                     | API                                                    |
| --------------- | ----------------------------------------------- | ------------------------------------------------------ |
| **CryptoTrade** | Cryptocurrency analysis and trading application | [Binance Testnet](https://testnet.binance.vision/)     |
| **ShopFront**   | Online store with catalog, cart, and checkout   | [commercetools](https://docs.commercetools.com/api)    |
| **MusicFlow**   | Music streaming service with search and player  | [Jamendo API](https://developer.jamendo.com/v3.0/docs) |

Detailed descriptions of each project, sprint requirements, and "Beyond API" features: **[PROJECTS.md](./PROJECTS.md)**.

The choice is locked in the repository README during Sprint 1. Changing the project after Sprint 1 is not allowed.

## 2. Team Formation

- **Team size:** 2 to 3 people. Recommended size — 3.
- **Team Lead:** Chosen by the team. Creates the project repository on GitHub and grants access to the other members.
- **Communication channels:** The team chooses its own tools (Telegram, Discord, etc.).

## 4. Using AI for Writing Code

In this project **using AI is allowed** for writing code. The key is transparency and honesty: reflect this in your development diary and clearly indicate which parts you wrote yourself and which were generated by an LLM agent.

## 5. Development Diary

Keeping a development diary is mandatory for this project, and it must be filled in on time.

It can be your story about learning Angular: which concepts you encounter, how you figure out DI or Change Detection, what you do with AI tools, and what you write yourself.

We care not only about the result but also the journey. The diary must be maintained during development — this is confirmed by the git commit history.

**Frequency:** at least 1 entry per week. Sprint 1 (1 week) — 1 entry, Sprints 2–4 (2 weeks each) — 1 entry.

More details: [DEVELOPMENT_DIARY.md](./DEVELOPMENT_DIARY.md)

## 6. Feature Component Approach

**Feature Component** — a self-contained functional block that solves a specific user task (UI screen, service, module).

**Each student must personally develop at least 2 Feature Components.** More details: [DEVELOPMENT_DIARY.md](./DEVELOPMENT_DIARY.md)

### Feature Component Examples

| Category | Example               | Description                                                      |
| -------- | --------------------- | ---------------------------------------------------------------- |
| UI       | Auth (Login/Register) | Reactive Forms, validation, JWT, protected routes                |
| UI       | Dashboard             | Screen with charts, progress, OnPush Change Detection            |
| UI       | Product Catalog       | List with filters, pagination, search                            |
| UI       | Task Board            | Drag & Drop, CRUD operations, reactive updates                   |
| Forms    | Multi-step Form       | Complex form with FormArray, custom validators, dynamic controls |
| Service  | Auth Service          | JWT, refresh tokens, guards, interceptors                        |
| State    | State Management      | NgRx / Signals-based store for state management                  |
| Testing  | Test Suite            | Unit tests for components, services, and pipes                   |

## 8. Project Presentation

After 4 sprints the team gets **1 week** to prepare for the final team presentation (~30 min).

The whole team presents the project together: what was built, how the team worked, which Angular concepts were used, and a live demo.

## 10. FAQ

### Q: Is Angular mandatory?

Yes. This is a project for learning Angular. Using another framework is not allowed.

### Q: Which Angular version should I use?

Angular 20+ (latest stable version recommended). Using standalone components, new control flow (`@if`, `@for`, `@switch`), and signals is encouraged.

### Q: Can I use Angular Material or other UI libraries?

Yes.

### Q: Is a backend required?

Not required. If you want to write a backend, **only NestJS is allowed**. The project focus is on the frontend (Angular).

### Q: Can I get a certificate without completing the project?

No. Students must complete the final project to receive a certificate.
