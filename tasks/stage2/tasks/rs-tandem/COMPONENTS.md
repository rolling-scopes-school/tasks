# Tandem: Компонентная архитектура

- Проект разделен на **компоненты** (модули)
- Компонент - архитектурная единица проекта, которую студент берёт в работу. Например: Auth, Dashboard, AI Agent, Quiz Widget  
- Виджет - отдельный тип компонента — интерактивный UI-блок для тренировки. Например: True/False, Memory Game, Async Sorter
- Каждый участник команды работает над **своими компонентами** (минимум 2 компонента)
- Каждый участник команды выбирает компоненты по своему уровню и интересам
- Каждый участник команды может создавать свои типы компонентов, не ограничиваясь списком в этом документе.

### AI Agent ⭐⭐⭐⭐

**Описание:** Полноценный AI-интервьюер. Один сложный компонент, который включает несколько подсистем.

**Что входит в компонент:**
1. **Chat UI** — интерфейс чата
2. **LLM Backend** — интеграция с моделью
3. **Streaming** — реальное время ответов
4. **Context Manager** — управление историей диалога

**Функциональность:**
- Чат с AI-агентом
- Streaming ответов (эффект печатной машинки)
- Markdown rendering (код, списки)
- Память контекста в рамках сессии

**Технические требования:**
- **No "Magic" SDKs:** Логика общения с LLM на чистом JS/TS (без LangChain, Vercel AI SDK)
- **Custom Streaming:** `fetch` + `ReadableStream`
- **Context Awareness:** Агент помнит контекст беседы

**Рекомендация:** Один человек из команды берет этот компонент целиком.

### Widget Engine ⭐⭐⭐

**Описание:** Движок для загрузки виджетов по JSON-схеме. Нужен, если команда делает виджеты.

**Функциональность:**
- Получение схемы от backend
- Динамическая загрузка виджетов
- Общий интерфейс для всех виджетов
- Валидация ответов

---

### Виджеты (Интерактивные задания)

*Каждый виджет = отдельный компонент. Все виджеты JSON-driven.*

#### Легкие виджеты (⭐⭐)

| Виджет | Описание |
|--------|----------|
| **Quiz** | Вопрос с выбором ответа |
| **True/False** | Утверждение — правда или ложь? |
| **Code Completion** | Заполни пропуски в коде |
| **Code Ordering** | Расставь строки кода (Drag & Drop) |

#### Средние виджеты (⭐⭐⭐)

| Виджет | Описание |
|--------|----------|
| **Memory Game** | "Be the Garbage Collector" — кликай на объекты для удаления |
| **Stack Builder** | Собери Call Stack — Drag & Drop блоков функций |
| **Async Sorter** | Распредели задачи по очередям Event Loop |

> **Подробности:** См. [WIDGETS_EXPLAINED.md](./WIDGETS_EXPLAINED.md)

**Создание своих типов виджетов:**
- Вы можете расширить `WidgetConfig` в `src/types/index.ts`, добавив новый тип виджета (например, `'regex-builder'` или `'prototype-chain'`)
- Widget Engine должен поддерживать ваш новый тип через Strategy Pattern (см. [ARCHITECTURE_AND_API.md](./ARCHITECTURE_AND_API.md))
- Важно: новый виджет должен быть JSON-driven и соответствовать общей архитектуре
