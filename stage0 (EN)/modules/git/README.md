# [Stage#0.](../../) Introduction to Git Version Control System and GitHub Web Service

## In this module students need to:

1. Get a general understanding of the Git version control system.
2. Start working with Git using GitHub web service as an example.
3. Get familiar with terminology:
   - Repository: Local, Remote/Origin
   - Working Directory
   - Staging Area
4. Get familiar with basic commands (clone, add, commit, push, pull)
5. Get familiar with the concept of "branch" in git: creating new, merging, rebasing
6. Get familiar with Markdown, format a page
7. Create your own web page using github-pages

## Approximate module completion time

Theory: 3 hours
Practice: 4 hours

## Theory

1. [Git lecture recording](https://youtu.be/6i0Wg-Pohg8?list=PLzLiprpVuH8df24MzZp-l5QMsJWJbi9qP). _Note_: if it seemed complicated, you can watch the brief course from additional materials step 0 and come back
2. View [materials from the presentation](https://slides.com/anton_bely/saturday-talk#/2)
3. You need to watch practical video materials [Git Workshop](https://youtu.be/Dlr_E7WfA08) (0.5 hours)
4. For the practical assignment, familiarize yourself with [Markdown documentation](https://guides.github.com/features/mastering-markdown/) - you can briefly look at the capabilities and syntax, and dive deeper in practice (0.5 hours)
5. Short and clear material on how to set up github-pages in your repository: [link to MDN](https://developer.mozilla.org/ru/docs/Learn/Common_questions/Using_Github_pages#%D0%B7%D0%B0%D0%B3%D1%80%D1%83%D0%B7%D0%BA%D0%B0_%D1%84%D0%B0%D0%B9%D0%BB%D0%BE%D0%B2_%D0%BD%D0%B0_github), [official documentation](https://pages.github.com/)
6. You need to read and remember [RS School commit requirements](https://rs.school/docs/ru/git-convention)
7. [Video](https://www.youtube.com/watch?v=0lvKcqP3aP0) about how to work with gh-pages
