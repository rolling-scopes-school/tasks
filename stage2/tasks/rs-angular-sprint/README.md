# RS Angular Sprint

## 1. Цель проекта

**Цели проекта:**

- Изучить основы фреймворка Angular через разработку реального приложения.
- Освоить ключевые концепции: Components, Routing, Signals, Directives, Pipes, Forms, DI, Change Detection, HTTP, Testing.
- Получить опыт командной работы и код-ревью.
- Продемонстрировать, чему вы научились за 4 спринта.

**Вы создаёте Angular-приложение в своём стиле** — с той функциональностью, которая вам интересна. Примеры возможных реализаций:

- Task Manager / Kanban-доска,
- Блог-платформа с авторизацией,
- Интернет-магазин / каталог товаров,
- Dashboard с визуализацией данных,
- ваша идея.

Главное требование — приложение должно демонстрировать владение ключевыми модулями Angular, которые покрываются в 4 спринтах.

## 2. Формирование команды

- **Состав команды:** От 2 до 3 человек. Рекомендуемый размер — 3 человека.
- **Team Lead:** Выбирается командой. Создаёт репозиторий проекта на GitHub и выдаёт доступы остальным участникам.
- **Каналы коммуникации:** Инструменты для общения (Telegram, Discord и др.) команда выбирает сама.

## 4. Использование AI для написания кода

В этом проекте **разрешено использовать AI** для написания кода. Главное — прозрачность и честность: отразите это в дневнике разработки и честно укажите, какие части написали вы лично, а какие были сгенерированы LLM-агентом.

## 5. Дневник разработки

Для этого проекта обязательно ведение дневника разработки, заполняемого своевременно.

Это может быть ваш рассказ о том, как вы изучаете Angular: с какими концепциями сталкиваетесь, как разбираетесь с DI или Change Detection, что делаете с помощью AI-инструментов, а что пишете сами.

Нам интересен не только результат, но и путь к нему. Дневник должен вестись по ходу разработки — это подтверждается историей коммитов в git.

**Частота:** минимум 1 запись в неделю. Sprint 1 (1 неделя) — 1 запись, Sprints 2–4 (2 недели каждый) — 2 записи.

Подробнее: [DEVELOPMENT_DIARY.md](./DEVELOPMENT_DIARY.md)

## 6. Feature Component подход

**Feature Component** — самостоятельный функциональный блок, решающий конкретную задачу пользователя (UI-экран, сервис, модуль).

**Каждый студент должен лично разработать минимум 2 Feature Component.** Подробнее: [DEVELOPMENT_DIARY.md](./DEVELOPMENT_DIARY.md)

### Примеры Feature Components

| Категория | Пример                | Описание                                                             |
| --------- | --------------------- | -------------------------------------------------------------------- |
| UI        | Auth (Login/Register) | Reactive Forms, валидация, JWT, protected routes                     |
| UI        | Dashboard             | Экран с графиками, прогрессом, OnPush Change Detection               |
| UI        | Product Catalog       | Список с фильтрами, пагинацией, поиском                              |
| UI        | Task Board            | Drag & Drop, CRUD операции, реактивное обновление                    |
| Forms     | Multi-step Form       | Сложная форма с FormArray, кастомными валидаторами, dynamic controls |
| Service   | Auth Service          | JWT, refresh tokens, guards, interceptors                            |
| State     | State Management      | NgRx / Signals-based store для управления состоянием                 |
| Testing   | Test Suite            | Unit-тесты для компонентов, сервисов и pipes                         |

## 8. Презентация проекта

После 4 спринтов команда получает **1 неделю** на подготовку к финальной презентации. Презентация состоит из двух частей:

### Часть 1: Командная презентация (~30 мин)

Вся команда вместе рассказывает о проекте: что сделали, как работали, какие Angular-концепции использовали, демо приложения.

Подробнее: [PRESENTATION.md](./PRESENTATION.md)

### Часть 2: Соло (~20 мин на студента)

Каждый студент выступает индивидуально сразу после командной части: рассказывает о своём личном вкладе в проект и отвечает на вопросы жюри. Даёт **50 баллов**.

Подробнее: [PRESENTATION.md](./PRESENTATION.md#часть-2-соло-20-мин-на-студента)

## 10. FAQ

### Вопрос: Обязательно ли использовать Angular?

Да. Это проект для изучения Angular. Использование другого фреймворка не допускается.

### Вопрос: Какую версию Angular использовать?

Angular 20+ (рекомендуется последняя стабильная версия). Приветствуется использование standalone components, новых control flow (`@if`, `@for`, `@switch`), signals.

### Вопрос: Можно ли использовать Angular Material или другие UI-библиотеки?

Да.

### Вопрос: Нужен ли бэкенд?

Не обязательно. Если хотите написать бэкенд, **только NestJS разрешен**. Фокус проекта на фронтенде (Angular).

### Вопрос: Можно ли получить сертификат без выполнения проекта?

Нельзя. Для получения сертификата студенты должны выполнить финальный проект.
