# RS Tandem: Гайд по оцениванию

> **Максимальный балл:** 800 pts + бонусы  
> **Для кого:** Студенты и Жюри

---

## 1. Философия

**Качество важнее количества.** Лучше 2 компонента сделать отлично, чем 5 "на троечку".

**Наша цель:** Оценить инженерную зрелость решения, глубину понимания технологий и **личный прогресс студента**.

### Состав Жюри

Жюри формируется **случайным образом** из пула менторов курса:

* **2 Ментора курса**
* **1 Активист RS School**
* **Итого:** 3 человека в жюри

---

## 2. Пороги допуска (Pass/Fail)

**Перед начислением баллов проверяются обязательные требования:**

| Требование | Как проверить |
|------------|---------------|
| TypeScript `strict: true` | `tsconfig.json` |
| Отсутствие `any` (или минимум с обоснованием) | Поиск по коду |
| **2 личных компонента** описаны в дневнике | [DEVELOPMENT_DIARY.md](./DEVELOPMENT_DIARY.md) |
| Приложение деплоится и работает | Ссылка на деплой |
| Дневник ведётся регулярно (не в последний день) | История коммитов |

**Если любой порог не пройден → Личная оценка = 0 баллов, допуск к защите не предоставляется.**

## 4. Личная оценка (500 pts)

### 4.1 Code Standards — 100 pts

**Документ:** [CODE_STANDARDS.md](./CODE_STANDARDS.md)

Объективно проверяемые критерии качества кода:

| Критерий | Баллы |
|----------|-------|
| TypeScript Strict | 25 |
| No Any | 25 |
| Linting Setup (ESLint + Prettier) | 25 |
| Error Handling | 25 |
| **Итого** | **100** |

### 4.2 Feature Checklist — 300 pts (ceiling)

**Документ:** [FEATURE_CHECKLIST.md](./FEATURE_CHECKLIST.md)

Студент выбирает фичи из каталога. Общая сумма может превышать 300, но ceiling = 300 pts.

**Примеры фич:**
- (+10) Unit Tests 20%+ coverage
- (+15) AI Chat basic
- (+25) AI RAG (embeddings + vector search)
- (+10) Theme Switcher
- (+15) WebSockets / Real-time sync

> **Важно:** Это "меню", а не обязательный список. Выбирайте то, что релевантно вашему проекту.

### 4.3 Expert Quality — 100 pts

**Субъективная оценка жюри** по двум критериям:

| Критерий | Баллы | Описание |
|----------|-------|----------|
| Technical Execution | 0-50 | Насколько хорошо реализовано технически |
| Product & UX | 0-50 | Насколько приятно пользоваться |
| **Итого** | **0-100** | |

#### Technical Execution (0-50 pts)

| Уровень | Баллы | Описание |
|---------|-------|----------|
| **Poor** | 0-15 | Работает, но много костылей. Код сложно читать. |
| **Good** | 15-35 | Хорошая реализация. Понятная архитектура. Код читаемый. |
| **Excellent** | 35-50 | Отличная реализация. Паттерны проектирования. Оптимизация. |

#### Product & UX (0-50 pts)

| Уровень | Баллы | Описание |
|---------|-------|----------|
| **Poor** | 0-15 | Базовый функционал. UX "сырой". Нет обратной связи. |
| **Good** | 15-35 | Работает как задумано. Loading states. Интуитивно понятно. |
| **Excellent** | 35-50 | "Вау-эффект". Анимации. Продуманные детали. |

## 6. Personal Breakthrough Bonus (+0-50 pts)

Дополнительные баллы за личный прогресс, которые **добавляются сверх** основной оценки.

| Баллы | Уровень | Описание |
|-------|---------|----------|
| **0** | Baseline | Работа соответствует текущему уровню студента |
| **+25** | Good Progress | Изучил новую технологию (backend, AI, testing) или ментор отмечает высокую дисциплину |
| **+50** | Heroic Shift | Огромный скачок. Пришёл слабым — сдал рабочий проект с AI. Вытянул задачи выбывшего напарника. |

**Как работает:**
1. Ментор заранее пишет Feedback о стартовом уровне студента
2. На защите студент рассказывает Learning Story
3. Жюри решает о начислении бонуса

## 7. Итоговая формула

```
Итоговый балл = Code Standards (100) + Features (ceiling 300) + Expert (100) + Team (300) + Bonus (0-50)

Максимум: 800 pts + 50 bonus = 850 pts
```

### Шкала оценки

| Баллы | Оценка |
|-------|--------|
| 720+ (90%+) | Отлично |
| 560-719 (70-89%) | Хорошо |
| 400-559 (50-69%) | Удовлетворительно |
| < 400 | Не зачтено |

## 8. Процедура защиты

### Формат

Защита проходит **по командам**, но с **личными слотами** для каждого студента.

### Структура слота (команда из 3 человек — 25 минут)

| Этап | Время | Описание |
|------|-------|----------|
| **Team Intro** | 2 мин | "Мы команда X, делали проект Y. Наша киллер-фича — Z." |
| **Student 1** | 5 мин | Demo компонентов + Learning Story |
| **Q&A Student 1** | 1 мин | Вопросы жюри |
| **Student 2** | 5 мин | Demo компонентов + Learning Story |
| **Q&A Student 2** | 1 мин | Вопросы жюри |
| **Student 3** | 5 мин | Demo компонентов + Learning Story |
| **Q&A Student 3** | 1 мин | Вопросы жюри |
| **Общие вопросы** | 2 мин | Вопросы по проекту в целом |
| **Deliberation** | 3 мин | Обсуждение оценок |

**Итого:**
* Команда из 3 человек: 25 минут
* Команда из 2 человек: 20 минут

## 10. Red Flags (Штрафы)

Если жюри видит эти проблемы, баллы снижаются:

| Red Flag | Где вычитается |
|----------|----------------|
| API Keys в коде | Code Standards: -25 pts (Error Handling = 0) |
| Обильное использование `any` | Code Standards: No Any = 0 |
| Нет обработки ошибок | Code Standards: Error Handling = 0 |
| Огромные файлы (500+ строк) | Expert Quality: Technical -10 |
| Copy-paste код | Expert Quality: Technical -10 |
