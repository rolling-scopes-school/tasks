## Условия получения

Все условия выполнены → 20 баллов, любое не выполнено → 0.

1. **Деплой** — приложение доступно по деплой-ссылке из README или запускается локально по инструкции из README.
2. **README-артефакты** — в `README.md` проекта в ветке `main` присутствуют все пункты:
   - О проекте (название, описание)
   - Команда (участники со ссылками на дневники)
   - Доска (ссылка + скриншот)
   - Лучшие PR (3-4 PR с содержательным code review)
   - Meeting Notes (ссылки на 3+ записей)
   - Feature List (3-5 фич с шагами воспроизведения)
   - На что обратить внимание в презентации (2-10 предложений)
   - Демо-видео (до 8 мин, основной user path приложения)
3. **Self-assessment PR** — создан, замержен в `main`, содержит ссылку на PR.
4. **Мерж дневника** — все записи замержены в `main`. Создан файл `username-merged.md` в `/development-notes/github-username/`.
5. **Регистрация на Peer Review** — заполнена Google-форма (ссылка в Discord).

> **Все три защиты обязательны.** Без командной защиты, peer review и менторской сессии — 0 баллов за соответствующий Score.
