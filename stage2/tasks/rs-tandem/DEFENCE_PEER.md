# Peer Review

Студент презентует свою работу 3 рандомно назначенным студентам-экспертам из других команд. ~30 мин.

**Фокус:** что сделано, работает ли, соответствует ли self-assessment.

Все сессии **открыты** — любой студент или ментор может присоединиться как наблюдатель.

## Формат сессии (~30 мин)

| Этап             | Время   | Что происходит                                               |
| ---------------- | ------- | ------------------------------------------------------------ |
| **Контекст**     | 2-3 мин | Студент представляет проект и свою роль.                     |
| **Feature Demo** | 20 мин  | Студент проходится по self-assessment, показывает фичи live. |
| **Q&A**          | 5 мин   | Эксперты: "Покажи X", "Что будет если сделать Y?"            |
| **Оценка**       | 2 мин   | Эксперты заполняют Google Form.                              |

## Принцип валидации

Всё заявленное в self-assessment **считается подтверждённым по умолчанию**. Эксперты отмечают только **расхождения** — фичи, которые не работают, выглядят завышенными или не были продемонстрированы. Если расхождений нет — значит self-assessment подтверждён.

## Оценка

- Каждый из 3 экспертов ставит оценку независимо (0-100).
- Финальная оценка = **медиана** трёх оценок.
- Если разброс > 40 баллов — координатор проверяет.
- Критерии и баллы: [SCORE_PERSONAL.md](./SCORE_PERSONAL.md#3-peer-review-100-баллов) (100 pts в RS APP).

### Google Form (заполняет каждый эксперт)

Форма **приватная** — студент не видит кто что написал.

1. **Deploy & Launch** (0-15) — работает ли деплой/локальный запуск
2. **Feature Verification** (0-50) — заявленное соответствует реальности
3. **Product Quality** (0-20) — user flow, edge cases (404, loading, ошибки)
4. **README & Artifacts** (0-15) — деплой-ссылка, Feature List, видео, self-assessment PR
5. **Что запомнилось из показанного** — свободное текстовое поле
6. **Расхождения с self-assessment** — какие фичи не работали, выглядели завышенными или не были продемонстрированы

## Подготовка студента

1. **Self-assessment PR** — создан до сессии ([подробности](./SCORE_PERSONAL.md#self-assessment))
2. **Подготовить демо** каждой заявленной фичи live
3. **Деплой** или быть готовым запустить локально

## Роль студента-эксперта

Каждому эксперту назначены **3 студента** для проверки (из разных команд, ~90 мин total).

1. **До сессии (~5 мин на студента):** прочитать self-assessment PR и README
2. **Во время сессии:** смотреть демо, отмечать расхождения с self-assessment
3. **После сессии:** заполнить Google Form
