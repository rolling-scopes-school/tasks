# Peer Review

Студент презентует свою работу 3 рандомно назначенным студентам-экспертам из других команд. ~15 мин.

**Фокус:** что сделано, работает ли, соответствует ли self-assessment.

Все сессии **открыты** — любой студент или ментор может присоединиться как наблюдатель.

## Формат сессии (~15 мин)

| Этап             | Время | Что происходит                                    |
| ---------------- | ----- | ------------------------------------------------- |
| **Контекст**     | 2 мин | Студент представляет проект и свою роль.          |
| **Feature Demo** | 8 мин | Студент показывает фичи из self-assessment live.  |
| **Q&A**          | 3 мин | Эксперты: "Покажи X", "Что будет если сделать Y?" |
| **Оценка**       | 2 мин | Эксперты заполняют скоркарды.                     |

Peers **подтверждают или корректируют** заявленные баллы из [self-assessment](./SCORE_PERSONAL.md#self-assessment).

## Оценка

- Каждый из 3 экспертов ставит оценку независимо (0-100).
- Финальная оценка = **медиана** трёх оценок.
- Если разброс > 40 баллов — координатор проверяет.
- Критерии и баллы: [SCORE_PERSONAL.md](./SCORE_PERSONAL.md#3-peer-review-100-баллов) (100 pts в RS APP).

## Подготовка студента

1. **Self-assessment PR** — создан до сессии ([подробности](./SCORE_PERSONAL.md#self-assessment))
2. **Подготовить демо** каждой заявленной фичи live
3. **Деплой** или быть готовым запустить локально

## Роль студента-эксперта

Каждому эксперту назначены **3 студента** для проверки (из разных команд, ~45 мин total).

1. **До сессии (~5 мин на студента):** прочитать self-assessment PR и README
2. **Во время сессии:** верифицировать заявленные фичи по живому демо
3. **После сессии:** заполнить скоркарду (Google Form)

> Чеклист продуктовых проверок: [CODE_STANDARDS.md](./CODE_STANDARDS.md)
