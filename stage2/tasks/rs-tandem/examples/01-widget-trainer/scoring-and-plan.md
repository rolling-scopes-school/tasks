# Scoring & Plan: Widget Trainer

Этот документ содержит пример расчета баллов и детальный план работ для команды из 3 человек.

## Зачем этот документ

Этот раздел — **образец планирования** для команд и менторов. Он демонстрирует:

1. **Базовый стек = хорошие баллы.** Vanilla TypeScript без фреймворков и AI-интеграции дает 655-680 баллов.
2. **Реалистичные сроки.** Эстимейты включают буфер +30-40% на изучение новых технологий.
3. **Процессы важны.** Митинги, код-ревью, дневник с первого дня — не бюрократия, а инвестиция в Teamwork Score.
4. **Cut Scope Strategy.** Заранее определено, что вырезать, если не успеваем.

> **Как использовать:** Это шаблон, не догма. Адаптируйте под свою команду, но сохраняйте принципы: планирование, коммуникация, документирование.

## Cut Scope Strategy (MoSCoW)

Заранее определите, что вырезать, если не успеваете. Это спасет вашу оценку.

### Must Have (обязательно)

- Auth (Login/Register)
- Dashboard (базовый)
- Library (список тем)
- Widget Engine (Quiz, True/False)
- 1 сложный виджет (Async Sorter ИЛИ Memory Game)
- Unit Tests 20%+
- E2E Tests (3 сценария минимум)
- README с описанием и скриншотами

### Should Have (желательно)

- 2-й сложный виджет
- Theme Switcher
- Responsive design
- i18n (2 языка)
- Audio API
- Firestore Security Rules
- Empty States

### Could Have (опционально)

- A11y для D&D (+10 баллов)
- Unit Tests 50%+
- Code Completion Widget
- Code Ordering Widget
- Анимации успеха/ошибки

### Won't Have (откладываем)

- AI интеграция
- 3D визуализации
- Realtime multiplayer
- OAuth (Google/GitHub) — Firebase Auth email достаточно

7. **Этот план — шаблон, не догма.** Адаптируйте сроки под свою команду, но сохраняйте принципы: планируйте, общайтесь, документируйте.
