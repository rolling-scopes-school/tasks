# RS Tandem: Teamwork

> **Score в RS APP:** Tandem: Teamwork  
> **Тип оценки:** на проект (одинаковая для всей команды)  
> **Формат:** Командная презентация (~25 мин)

## Критерии оценки командной работы (макс. 300 баллов)

Эта оценка одинакова для всех участников команды.

### 1. Project Management & Communication (100 баллов)

> **Важно:** Ведение доски, создание Issues и документирование встреч — **ответственность всей команды**, а не одного человека. Оценивается, насколько каждый участник вовлечён в процессы.

| Критерий               | Баллы  | Описание требований                                                                                                                                                                                                |
| ---------------------- | ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Active Board**       | **40** | **Использование Kanban/Scrum доски** (GitHub Projects, Trello, Jira).<br>✅ Доска живая: задачи двигаются, а не созданы в один день перед сдачей.<br>✅ История изменений охватывает минимум 3 недели разработки.  |
| **Task Decomposition** | **30** | **Работа с Issues.**<br>✅ Cкоуп проекта декомпозирован на тикеты.<br>✅ У каждого Issue есть понятное описание, Assignee и Label (bug, feature, refactor). Есть минимум 3 тикета на каждого участника команды     |
| **Sync Culture**       | **30** | **Регулярные синки.**<br>✅ Минимум 3 задокументированных Meeting Notes. Каждый должен содержать: краткие итоги + принятые решения с ответственными.<br>✅ Ссылки на эти записи должны быть указаны в `README.md`. |

## Формат презентации (~25 мин)

Презентации проходят блоками по 5 команд. Жюри знакомится с README и артефактами (доска, PR, видео) **заранее** — до начала сессии.

| Часть            | ~Время | Что происходит                                                                                 |
| ---------------- | ------ | ---------------------------------------------------------------------------------------------- |
| **Product Demo** | 12 мин | Команда показывает приложение (live или видео) и подсвечивает технические решения под капотом. |
| **Team Story**   | 5 мин  | Как работали: роли, процессы, конфликты, решения.                                              |
| **Q&A**          | 8 мин  | Вопросы жюри по архитектуре, коду и нестандартным решениям.                                    |

> По желанию и возможностям жюри времени на команду может быть предоставлено больше.

После каждого блока жюри номинирует лучших в категориях Awards (Best Application, Engineering Excellence, Best AI Integration, Design Excellence, Best Presentation).

> **Зачем Team Story:** 6 недель работы — это истории про конфликты, решения, компромиссы. Это интереснее для жюри, чем листание Kanban-доски. И именно здесь видно, была ли реальная командная работа.

## Что нужно от команды (в README до дедлайна)

1. **О проекте:** Название и краткое описание приложения.
2. **Команда:** Список участников со ссылками на их Дневники разработки.
3. **Доска:** Ссылка на Kanban/Scrum доску + скриншот.
4. **Лучшие PR:** 3-4 Pull Request с содержательным ревью.
5. **Meeting Notes:** Ссылки на 3+ записей со встреч.
6. **Деплой:** Ссылка на работающее приложение или инструкция для локального запуска.
7. **Feature List:** 3-5 фич с шагами (для кросс-чека продукта).
8. **Чем гордимся:** 2-10 предложений о сильных сторонах проекта. На что обратить фокус (ориентир для жюри).
9. **Демо-видео (до 8 мин)** — запись основного user path приложения. Обязательно для всех команд.
