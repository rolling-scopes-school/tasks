# RS Tandem: Teamwork 

## Формирование команды

* **Состав команды:** От 2 до 6 человек из числа тех, кто пишет код. Менторы могут писать код и быть полноценными участниками. Мы ожидаем, что ментора будут заниматься разработкой back-end или разработкой AI агента.
* **Team Lead:** Выбирается командой путем голосования. Создает репозиторий проекта на GitHub и выдает доступы остальным участникам.
* **Регистрация:** Каждый участник регистрируется самостоятельно через форму https://forms.gle/UQNknowafoFj1hEt8
* **Каналы коммуникации:** Инструменты для общения (Telegram, Discord и др.) команда выбирает сама.

## Критерии оценки командной работы (макс. 300 баллов)

### 1. Project Management & Communication (100 баллов)
| Критерий | Баллы | Описание требований |
|----------|-------|---------------------|
| **Active Board** | **40** | **Использование Kanban/Scrum доски** (GitHub Projects, Trello, Jira).<br>✅ Доска живая: задачи двигаются, а не созданы в один день перед сдачей.<br>✅ История изменений охватывает весь период разработки (минимум 3 недели). |
| **Task Decomposition** | **30** | **Работа с Issues.**<br>✅ Весь скоуп работ декомпозирован на тикеты.<br>✅ У каждого Issue есть понятное описание, Assignee и Label (bug, feature, refactor).|
| **Sync Culture** | **30** | **Регулярные синки.**<br>✅ Минимум 3 задокументированных примера Meeting Notes (краткие итоги созвона + принятые решения).<br>✅ Ссылки на эти записи должны быть указаны в `README.md`. |

| Критерий | Баллы | Описание требований |

## Как презентовать командную работу

На защите у вашей команды будет **5-10 минут** на блок Teamwork.
Перед защитой мы просим вас собрать необходимую информацию в файле **`README.md`** вашего репозитория.

**Что должно быть в `README.md`:**

1.  **О проекте:** Название и краткое описание приложения.
2.  **Команда:** Список участников со ссылками на их Дневники разработки (`DEVELOPMENT_DIARY.md`).
3.  **Деплой:** Ссылка на работающее приложение.
4.  **Процессы:**
    * Скриншот вашей Kanban/Scrum доски + ссылка на нее.
    * Ссылки на 3-4 **лучших Pull Request**, демонстрирующих ваш Code Review процесс.
5. Любая другая релевантная информация
