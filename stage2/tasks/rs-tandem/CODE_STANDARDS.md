# Code Standards (100 баллов)

> **Score в RS APP:** Tandem: Code  
> **Тип оценки:** на проект (одинаковая для всей команды)  
> **Формат:** Cross-Check

---

## Cross-Check Requirements (+100)

#### 1. TypeScript Strict (+25)

1. (+10) TypeScript используется, в `tsconfig.json` включён `"strict": true`.
2. (+10) В коде нет `@ts-ignore`.
3. (+5) В коде нет `as any` (приведение к any для обхода типизации).

**Как проверить:** открыть `tsconfig.json`, затем выполнить поиск по проекту:
```bash
grep -r "@ts-ignore\|as any" src/
```

#### 2. No Any (+25)

1. (+25) В коде нет явного использования типа `any` (`: any`, `<any>`).
2. (-10) 1-10 использований `any` — частичный балл (15 из 25).
3. (-25) Более 10 использований `any` — 0 баллов за эту секцию.

**Как проверить:**
```bash
grep -rn ": any\|<any>" src/ --include="*.ts" --include="*.tsx" | wc -l
```

> **Исключение:** `any` в типах внешних библиотек (если нет `@types`) — не штрафуется при наличии комментария `// TODO: replace any`.

#### 3. Linting Setup (+25)

1. (+10) ESLint настроен: файл конфигурации `.eslintrc.*` или `eslint.config.*` существует, `npm run lint` выполняется без ошибок.
2. (+10) Prettier настроен: файл `.prettierrc` или `prettier.config.*` существует, код отформатирован единообразно.
3. (+5) Husky настроен: pre-commit hook запускает lint/format.

**Как проверить:**
```bash
npm install && npm run lint
```
Убедиться, что нет ошибок линтинга. Проверить наличие файлов конфигурации.

#### 4. Error Handling (+25)

1. (+10) API-запросы обёрнуты в try/catch. Приложение не падает при ошибках сети.
2. (+10) Есть UI для отображения ошибок (toast, error page, alert). Сообщения понятны пользователю.
3. (+5) Приложение корректно работает при отключении сети: показывает сообщение, не "белый экран".

**Как проверить:** открыть приложение, отключить сеть (DevTools → Network → Offline), выполнить действие, требующее сети.
