# Менторская сессия

Индивидуальный созвон студента с рандомно назначенным ментором. **Чисто технический** фокус: код, архитектура, понимание.

Ментор **не проверяет "работает ли"** — это уже сделали студенты-эксперты на [Peer Review](./DEFENCE_PEER.md). Ментор фокусируется на коде.

## Формат (~30 мин)

| Этап            | Время  | Описание                                                                                                   |
| --------------- | ------ | ---------------------------------------------------------------------------------------------------------- |
| **О проекте**   | 3 мин  | Студент даёт контекст: что за приложение, его роль в команде.                                              |
| **Дневник**     | 5 мин  | Студент кратко рассказывает о своём пути: что изучил, какие проблемы решал. Ментор просматривает дневник.  |
| **Code Review** | 15 мин | Студент показывает код фич со ★ из self-assessment и объясняет архитектуру. Ментор задаёт вопросы по коду. |
| **Q&A**         | 5 мин  | Вопросы по архитектуре, техническим решениям и чекпоинтам (W1-W5).                                         |
| **Оценка**      | 2 мин  | Ментор заполняет Google Form с обоснованием.                                                               |

## Оценка

Критерии и баллы: [SCORE_PERSONAL.md](./SCORE_PERSONAL.md#4-mentor-review-150-баллов) (150 pts).

Ментор **может снизить баллы**, если студент не понимает свой код или не может объяснить архитектурные решения.

## Подготовка студента

1. **Self-assessment PR** должен быть создан ([подробности](./SCORE_PERSONAL.md#self-assessment))
2. **Дневник** замержен в `main` (файл `username-merged.md`)
3. **Подготовить "О проекте"** — 2-3 предложения: что за приложение, ваша роль
4. **Знать код** фич со ★ — где что лежит, уметь объяснить архитектуру
5. **Быть готовым к вопросам** по коду, техническим решениям и чекпоинтам (W1-W5)

## Подготовка ментора

1. Ознакомиться с критериями [Mentor Review](./SCORE_PERSONAL.md#4-mentor-review-150-баллов)
2. Перед созвоном (~5 мин): открыть self-assessment PR, посмотреть оценку и замечания Peer Review (если есть расхождения — обратить внимание на эти фичи)
3. Бегло просмотреть репозиторий и дневник
4. На созвоне: **фокус на коде** — продукт уже проверен peers, не нужно тестировать user flow
5. После созвона: заполнить Google Form
