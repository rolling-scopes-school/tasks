# Custom Video Player

## Skills

`DOM API` `HTMLMediaElement` `Fullscreen API` `keyboard events` `range input` `CSS variables`

## Task Description

**Custom Video Player** is a video player whose look-and-feel is defined entirely by the designer rather than by the browser's default controls. A custom player lets a project keep its visual identity inside any app, game, or marketing site. The native `<video>` element is used as the engine, while the play/pause, volume, scrubber, speed and fullscreen controls are built in HTML/CSS/JS on top of it.

## Mandatory Additional Feature

Implement **keyboard control** of the player, matching how the corresponding keys behave in YouTube:

1. `Space` — toggle play / pause.
2. `M` — toggle mute / unmute.
3. `>` — increase playback rate.
4. `<` — decrease playback rate.
5. `F` — toggle fullscreen mode.

Hotkeys must work when the player has focus (or whenever the player is the only one on the page), and must not fire while the user is typing into an input field.

## Optional Improvements

Each well-executed item below is worth **+10** points (Stage 3 in [js30.md](js30.md), capped at 30 per widget). You may also invent your own improvements of comparable complexity.

- Support **additional YouTube-style hotkeys** — arrow keys for seek and volume, `J` / `K` / `L`, `0`–`9` for percent-jump, `,` / `.` for frame stepping, `C` for captions. The full list is shown by `Shift + /` on any YouTube video. **Each additional hotkey awards +2 points**, accumulating within the Stage 3 cap.
- A **video carousel / slider** — switch between several video sources with prev / next controls and a thumbnail strip. [Reference](https://50projects50days.com/projects/background-slider/) (use videos instead of images).
- A **mini movie catalog** — a "kinopoisk" of your own where each tile opens a video in the player. [Reference](https://50projects50days.com/projects/movie-app/) (use videos instead of images).
