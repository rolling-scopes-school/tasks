# Whack-A-Mole

## Skills

`DOM API` `mouse events` `setTimeout` / `setInterval` `localStorage` `CSS animations` `game loop basics`

## Task Description

**Whack-A-Mole** is a reaction-time game: moles randomly pop up out of holes for a short time, and the player scores points by clicking them before they disappear. The round ends after a fixed time and a final score is shown.

![Whack-A-Mole screenshot](images/js30-6.jpg)

- [Reference demo](https://irinainina.github.io/JavaScript30-1/30%20-%20Whack%20A%20Mole/index-FINISHED.html)
- [Author video walkthrough](https://youtu.be/toNFfAaWghU) (14:35)
- Parent task and scoring: [js30.md](js30.md)

## Mandatory Additional Feature

Extend the game with **progressively harder levels**, **persist the current level and best score in `localStorage`**, and display the saved values on the game screen after a page reload. The number of levels and the difficulty model (faster moles, more holes active simultaneously, decoy targets that subtract points, shorter pop-up time, etc.) are up to you ‚Äî pick a model that actually gets harder and document it briefly in the in-app help or the README.

## Optional Improvements

Each well-executed item below is worth **+10** points (Stage 3 in [js30.md](js30.md), capped at 30 per widget). You may also invent your own improvements of comparable complexity.

- Add sound effects (hit, miss, level-up, game-over) with a mute toggle.
- Cap the number of clicks per round so the player has to think before whacking.
- Polished mole animations: spawn / hide easing, hit reaction, particle burst.
- Randomise the moles' on-screen position rather than fixing them to a static grid.
- Combining all of the above pushes the project toward [Duck Hunt](https://nimlu-bot-rsclone.netlify.app/) ‚Äî one of the best final projects of the 2020Q3 set. See the author's [post-mortem article](https://andreykomov.medium.com/must-read-310726007e79).
- Replace mouse-click with **keyboard mode** ‚Äî each hole is mapped to a key and the game becomes a typing/reaction trainer.
- A high-scores table stored in `localStorage` with player names.
